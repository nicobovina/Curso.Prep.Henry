En Javascipt se llama prototype a una propiedad la cual un objeto (vamos a llamarlo Objeto1) hace referencia a otro objeto. Este otro objeto (objeto prototipo) contiene la información sobre las propiedades y métodos del Objeto1. Se hace uso de esta propiedad, dado que si crearamos N-Objetos1 con todas las propiedades y métodos ocuparía mucho espacio en memoria (se pierde eficiencia). Por lo que cuando creamos Objeto1, procuramos crearlo de la forma más eficiente posible (es decir, creando lo que se necesita solamente). No obstante, Objeto1 cuenta con la posibilidad de utilizar las propiedades y métodos del Objeto Prototipo, dado que se encuentran enlazados por la propiedad prototype.

Se llama Constructor al método que se utiliza para crear e inicializar un objeto a partir de una clase. Los argumentos del método pueden ir de 0 a N-argumentos, dependiendo si se crea a partir de una clase base (en este caso no se le pasan argumentos), o se deriva de otra clase (en este caso toma N-argumentos).
